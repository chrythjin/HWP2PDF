- ✅ 모든 설정값에 적절한 기본값 제공

#### [routes/v1.ts](file:///c:/NEW%20PRG/HWP2PDF/apps/api/src/routes/v1.ts) — API 라우트 핸들러
- ✅ `POST /v1/uploads/initiate` — GCS 직접 업로드 URL 발급 (Signed URL v4)
- ✅ `POST /v1/uploads/complete` — 직접 업로드 완료 후 변환 시작
- ✅ `POST /v1/upload` — 멀티파트 폴백 업로드
- ✅ `GET /v1/jobs/:jobId` — 작업 상태 폴링
- ✅ `GET /v1/results/:fileName` — 로컬 모드 PDF 다운로드
- ✅ 모든 핸들러에서 입력 검증 후 `next(ApiError)` 패턴 일관 적용
- ✅ `202 Accepted` + `Location` 헤더로 비동기 작업 표준 패턴 준수

> [!TIP]
> `/v1/results/:fileName` 라우트(L196)는 하드코딩된 경로 문자열입니다. `API_ROUTES`에 `RESULTS` 상수를 추가하면 일관성이 향상됩니다.

#### [services/conversion-service.ts](file:///c:/NEW%20PRG/HWP2PDF/apps/api/src/services/conversion-service.ts) — LibreOffice 변환
- ✅ 변환 완료 후 원본 파일 자동 삭제 (`finally` 블록)
- ✅ LibreOffice 프로세스당 격리된 사용자 프로필 디렉토리 생성 → 동시 변환 지원
- ✅ 프로필 디렉토리도 `finally`에서 정리
- ✅ 변환 실패 시 stderr 메시지 전달

> [!IMPORTANT]
> **타임아웃 미설정**: `runLibreOffice`에 타임아웃이 없습니다. 악의적이거나 손상된 HWP 파일이 LibreOffice를 무한 행(hang)시킬 경우 프로세스가 영원히 대기합니다. 프로덕션 배포 시 `setTimeout` + `process.kill()`로 최대 변환 시간(예: 2분)을 제한하는 것을 **강력히 권장**합니다.

#### [services/job-store.ts](file:///c:/NEW%20PRG/HWP2PDF/apps/api/src/services/job-store.ts) — 작업 저장소
- ✅ `JobStore` 인터페이스로 추상화 → `MemoryJobStore` / `FirestoreJobStore` 전환 가능
- ✅ 만료 감지 로직(`isExpired`)이 읽기 시점에 lazy 평가 — 별도 스케줄러 불필요
- ✅ Firestore 사용 시 `projectId`, `databaseId` 설정 지원

> [!NOTE]
> `FirestoreJobStore.updateJob`(L118-132)에서 `get()` → `set()` 패턴은 동시 업데이트 시 마지막 쓰기 우선(last-write-wins)입니다. 현재 단일 작업당 단일 변환 프로세스만 업데이트하므로 실질적 문제는 없지만, 트랜잭션으로 감싸면 더 안전합니다.

#### [services/storage-service.ts](file:///c:/NEW%20PRG/HWP2PDF/apps/api/src/services/storage-service.ts) — 스토리지 서비스
- ✅ `shouldUseGcs()` 조건에 따라 로컬/GCS 모드 분기 처리
- ✅ Signed URL v4로 업로드/다운로드 URL 생성
