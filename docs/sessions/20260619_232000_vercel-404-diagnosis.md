# Session Note: Vercel 404 NOT_FOUND Diagnosis

## 1. Overview

Production URL `https://hwp2pdf-phi.vercel.app` returns Vercel `404: NOT_FOUND`. Cloud Run backend API is healthy and serving requests. Root cause is on the Vercel deployment / project-configuration side, not the API or the application code.

---

## 2. Verified facts

### Direct observations (wire-level)

- `GET https://hwp2pdf-phi.vercel.app/` → `404 Not Found`
  - `Server: Vercel`
  - `X-Vercel-Error: NOT_FOUND`
  - Body: `The page could not be found / NOT_FOUND / icn1::...`
- `GET https://hwp2pdf-api-130439872251.asia-northeast3.run.app/health` → `200 OK`
- `POST https://hwp2pdf-api-130439872251.asia-northeast3.run.app/v1/uploads/initiate` → `201 Created` with a valid GCS signed URL
- `POST https://hwp2pdf-phi.vercel.app/v1/uploads/initiate` → `404 Not Found` (expected, no proxy configured)

### Local build verification

- `pnpm --filter web build` succeeds and prerenders `/` as static content.
- Route table output:
  - `/` ○ Static
  - `/_not-found` ○ Static

### GitHub Actions deployment history

- `deploy-api-cloud-run.yml` ran successfully on 2026-06-19 13:54 (workflow_dispatch) and deployed revision `hwp2pdf-api-00001-szd`; smoke test passed.
- `deploy-web-vercel.yml` has **no successful runs since 2026-06-15**.
- The last listed run (2026-06-15) ended in `conclusion: success` but the `preflight` job reported:
  - `Vercel deployment is not configured yet. Missing: VERCEL_TOKEN VERCEL_ORG_ID VERCEL_PROJECT_ID NEXT_PUBLIC_API_BASE_URL`
  - `configured=false`
  - Therefore the actual `deploy` job was skipped.

---

## 3. Root-cause assessment

1. The Cloud Run API is live and correctly configured.
2. The Next.js application code builds cleanly and produces a static `/` route.
3. The Vercel URL responds with a Vercel-native 404, which means the deployed Vercel project has no valid page/function to serve for `/`.
4. The GitHub Actions path that is supposed to deploy the frontend has never executed a real deployment because the required Vercel secrets/variables are missing.

Most likely causes on Vercel:

- **Project Root Directory is not set to `apps/web`**, so Vercel does not find the Next.js app and deploys an empty project.
- **Framework Preset** is not set to **Next.js**, or the **Build Command** does not install workspace dependencies and build from the monorepo root.
- **Output Directory** is not `.next`.
- A previous failed deployment is still the active production deployment, and a fresh redeploy with correct settings (and cleared build cache) has not been performed.

---

## 4. Required fixes

### A. Vercel Dashboard (immediate)

Open the `hwp2pdf-phi` project settings and verify:

- **Root Directory**: `apps/web`
- **Framework Preset**: `Next.js`
- **Build Command**:
  ```bash
  cd ../.. && pnpm install --frozen-lockfile && pnpm --filter @hwp2pdf/shared build && pnpm --filter web build
  ```
- **Output Directory**: `.next` (leave default)
- **Environment Variables**:
  - `NEXT_PUBLIC_API_BASE_URL=https://hwp2pdf-api-130439872251.asia-northeast3.run.app`

Then trigger a **Redeploy** and, if a previous failed build is cached, choose **Use Existing Build Cache = No**.

### B. GitHub Actions automation

Add the following repository secrets / variables at `Settings → Secrets and variables → Actions`:

- Secrets:
  - `VERCEL_TOKEN`
  - `VERCEL_ORG_ID`
  - `VERCEL_PROJECT_ID`
- Variables:
  - `NEXT_PUBLIC_API_BASE_URL=https://hwp2pdf-api-130439872251.asia-northeast3.run.app`

After they are set, pushes to `master` that touch `apps/web/**`, `packages/shared/**`, `package.json`, `pnpm-lock.yaml`, or `pnpm-workspace.yaml` will automatically deploy.

## 5. Verification steps after fix

1. Run `curl -I https://hwp2pdf-phi.vercel.app/` and confirm `HTTP/2 200`.
2. Open the browser, drop a small `.hwp` file, and confirm the upload initiates against the Cloud Run API.
3. Watch the job polling reach `completed` and the signed PDF download URL appear.
