## Key Decisions

1. **UI gating is convenience only**: Notice category hidden from non-admin users in write/edit forms and list filter. The API independently enforces the permission matrix.
2. **No author field in forms**: authorId/authorName are server-derived from the token. Tests verify these fields are absent from POST/PATCH request bodies.
3. **Plain text body rendering**: Post body rendered in `<pre>` with `whitespace-pre-wrap` — no `dangerouslySetInnerHTML`. XSS test confirms script tags display as text.
4. **Edit/delete visibility**: Controls shown when `isOwner || admin || boardModerator`.
5. **vi.hoisted for mock state**: Tests use `vi.hoisted` to create mutable refs that survive vitest's `vi.mock` hoisting, enabling the `@/lib/firebase` mock's `auth.currentUser` getter to return the correct test user.
