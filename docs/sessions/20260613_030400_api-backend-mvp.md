# 2026-06-13 API backend MVP

## 변경 요약

- `apps/api` Express + TypeScript 패키지를 추가했다.
- 공유 계약(`@hwp2pdf/shared`)의 `API_ROUTES`, 업로드 제한, 응답 타입을 사용해 다음 API를 구현했다.
  - `GET /health`
  - `POST /v1/upload`
  - `GET /v1/jobs/:jobId`
- `.hwp` 단일 파일 업로드, 20MB 제한, IP당 시간당 60회 rate limit, 기본 CORS/Helmet 설정을 추가했다.
- 로컬 임시 저장소(`tmp/uploads`, `tmp/results`)와 인메모리 job store를 구현했다.
- 변환 서비스는 `LIBREOFFICE_BIN` 또는 `soffice`를 실행하는 경계로 구현했다. 현재 Windows 개발 환경에는 LibreOffice가 없어 실제 변환 성공 대신 명확한 converter configuration 실패 메시지를 반환한다.

## 검증

- `pnpm --filter @hwp2pdf/shared build && pnpm --filter api build` 통과.
- `pnpm --filter api build` 통과.
- 임시 Node HTTP 드라이버로 실제 Express 앱을 띄워 surface를 확인했다.
  - `GET /health` -> `200 {"status":"ok"}`
  - 비-HWP 업로드 -> `422 invalid_file`
  - `sample.hwp` 업로드 -> `202 queued`
  - job status polling -> `200 failed` with `LibreOffice 변환 엔진을 실행할 수 없습니다...`
