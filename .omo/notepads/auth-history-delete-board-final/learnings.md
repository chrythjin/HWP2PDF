## 2026-06-23 F1 plan compliance audit
- Verdict: APPROVE. Every D1-D13 decision from `.omo/drafts/auth-history-delete-board-redesign.md` is implemented in source or documented output; no decision is deferred or missing based on this audit.
- Scope/evidence read: `.omo/plans/auth-history-delete-board-final.md`, `.omo/drafts/auth-history-delete-board-redesign.md`, `apps/api/src/routes/v1.ts`, `apps/api/src/middleware/auth.ts`, `apps/api/src/middleware/worker-auth.ts`, `apps/api/src/services/firebase-admin.ts`, `apps/api/src/services/job-store.ts`, `apps/api/src/services/cloud-tasks-dispatcher.ts`, `apps/api/src/routes/v1.upload-ownership.test.ts`, `apps/api/src/routes/v1.download-auth.test.ts`, `apps/api/src/routes/v1.board.ts`, `apps/web/src/hooks/useBoardClaims.ts`, `apps/web/src/app/board/page.tsx`, `packages/shared/src/index.ts`, `docs/plan/auth-history-delete-board-roadmap.md`, and `docs/INDEX.md`.
- D1 Auth approach: IMPLEMENTED. API auth middleware verifies Firebase Bearer ID tokens only and surfaces claims (`apps/api/src/middleware/auth.ts:1-17`, `auth.ts:82-130`); Firebase Admin uses ADC/service-account/mock verification (`apps/api/src/services/firebase-admin.ts:1-15`, `firebase-admin.ts:63-181`). API source grep found no password/login/signup/refresh matches under `apps/api/src`; shared contract explicitly forbids server password/login DTO exports (`packages/shared/src/index.ts:8-10`) and tests assert no `LoginRequest`, `SignUpRequest`, `AuthTokens`, or refresh routes (`packages/shared/src/index.test.ts:342-349`).
- D2 Firestore security boundary: IMPLEMENTED. The plan requires API authorization as the primary boundary. Verified route-level ownership/auth checks for status/download/upload/history/board in Express: owner verifier on job status (`apps/api/src/routes/v1.ts:378-416`), protected download (`v1.ts:434-497`), member history/delete behind `requireAuth` (`v1.ts:752-839`), and board routes behind `requireAuth` (`apps/api/src/routes/v1.board.ts:94-300`).
- D3 Firebase Admin credentials: IMPLEMENTED. Production default is ADC unless fallback env is present (`apps/api/src/config.ts:4-18`), with service-account fallback described as local/non-GCP only (`config.ts:40-51`) and implemented in Firebase Admin initialization (`apps/api/src/services/firebase-admin.ts:63-181`).
- D4 Durable conversion guarantee: IMPLEMENTED. Upload completion persists queued jobs and dispatches via `enqueueConversionJob`, replacing direct fire-and-forget conversion (`apps/api/src/routes/v1.ts:267-290`, `v1.ts:339-357`). Worker endpoint processes queued jobs with idempotent no-ops and retryable/terminal failure handling (`v1.ts:571-696`).
- D5 Anonymous deletion: IMPLEMENTED. Only member deletion endpoint exists under `/v1/me/jobs/:jobId` and requires `requireAuth` (`apps/api/src/routes/v1.ts:799-839`); anonymous upload/status paths issue/verify access tokens but no anonymous delete route was found in the verified route file.
- D6 Anonymous status/download access: IMPLEMENTED. Anonymous initiate creates a hashed token and returns plaintext once with `accessTokenHeader` (`apps/api/src/routes/v1.ts:145-190`); complete/status/download require `X-Job-Access-Token` through `extractAnonymousTokenFromHeaders` and owner verifier (`v1.ts:237-260`, `v1.ts:378-416`, `v1.ts:434-497`). Tests assert no token -> 401, wrong token -> 403, correct token -> 200 (`apps/api/src/routes/v1.upload-ownership.test.ts:386-451`, `v1.upload-ownership.test.ts:603-631`; `apps/api/src/routes/v1.download-auth.test.ts:105-168`, `v1.download-auth.test.ts:241-289`).
- D7 Member history retention: IMPLEMENTED. Job records contain separate `downloadExpiresAt` and `metadataExpiresAt` fields (`apps/api/src/services/job-store.ts:36-40`); member history response includes both and omits expired download URLs (`apps/api/src/routes/v1.ts:726-747`); listing returns only current user's non-deleted jobs (`v1.ts:752-761`).
- D8 Board admin model: IMPLEMENTED. Firebase custom claims `admin` and `boardModerator` are decoded in auth middleware (`apps/api/src/middleware/auth.ts:35-43`, `auth.ts:100-106`), board routes require member auth and enforce notice/admin plus moderator/admin edit/delete behavior (`apps/api/src/routes/v1.board.ts:94-300`). Frontend claim gating uses `user.getIdTokenResult(true)` (`apps/web/src/hooks/useBoardClaims.ts:40-56`) and board page login/admin convenience gating (`apps/web/src/app/board/page.tsx:30-78`, `board/page.tsx:93-153`).
- D9 Plan output target: IMPLEMENTED. Human-facing plan exists at `docs/plan/auth-history-delete-board-roadmap.md` with final plan content (`docs/plan/auth-history-delete-board-roadmap.md:1-17`) and `docs/INDEX.md` links it as the full final implementation plan (`docs/INDEX.md:37-40`).
- D10 Test strategy: IMPLEMENTED. Plan todos mark core API/security/worker tests and UI tests completed (`.omo/plans/auth-history-delete-board-final.md:123-225`). Verified required route tests for upload ownership and download auth exist and cover token boundary (`apps/api/src/routes/v1.upload-ownership.test.ts:24-37`, `v1.download-auth.test.ts:12-22`).
- D11 Durable queue technology: IMPLEMENTED. Dispatcher supports Cloud Tasks HTTP tasks with OIDC token service account/audience (`apps/api/src/services/cloud-tasks-dispatcher.ts:1-18`, `cloud-tasks-dispatcher.ts:218-280`), config selects cloud-tasks when queue env is present and inline only as local/dev fallback (`apps/api/src/config.ts:57-80`), and worker endpoint requires OIDC (`apps/api/src/middleware/worker-auth.ts:1-13`, `worker-auth.ts:190-250`).
- D12 Deletion metadata: IMPLEMENTED. Member delete deletes stored files then calls `markJobDeleted` (`apps/api/src/routes/v1.ts:823-835`); `markJobDeleted` sets `status: "deleted"`, `deletedAt`, `deletedBy`, `tombstoneUntil` using `TOMBSTONE_RETENTION_MS`, and clears `downloadUrl`, `resultObjectPath`, and `originalObjectPath` (`apps/api/src/services/job-store.ts:242-262`, `job-store.ts:443-463`).
- D13 Anonymous token transport: IMPLEMENTED. Route code imports and returns `ANONYMOUS_ACCESS_TOKEN_HEADER`, and all anonymous verification reads headers rather than query strings (`apps/api/src/routes/v1.ts:5-7`, `v1.ts:175-188`, `v1.ts:237-260`, `v1.ts:388-392`, `v1.ts:423-427`).
